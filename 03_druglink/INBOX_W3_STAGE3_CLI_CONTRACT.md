# W16 → W3: the exact Stage-3 v2 CLI contract

Stage-3's v2 CLI is wired and fail-closed. This is the exact surface it binds, so you can
generate bytes it will consume without a second round-trip.

## The invocation

```bash
python -m druglink.run_stage3 --v2 \
  --stage2-manifest       OUT/stage2_run_manifest.json \
  --stage2-report         OUT/stage2_aggregate_verification.json \
  --bundles-root          OUT/ \
  --stage1-release        <the authoritative Stage-1 v3 release FILE> \
  --stage2-bridge         OUT/stage3_bridge.json \
  --stage2-bridge-report  OUT/stage3_bridge_verification.json \
  --universe-store        <admitted universe store dir> \
  --artifact-class        analysis \
  --output-root           OUT_STAGE3/
```

Every one is REQUIRED on the `--v2` path. There is no default and no fixture fallback: a
missing input is a named refusal (exit 2), and **nothing is written**.

## What Stage-3 binds, and what it re-derives

| artifact | schema | Stage-3 does |
|---|---|---|
| aggregate manifest | `spot.stage02_run_manifest.v3_topology_only` | **recomputes** the semantic self-hash (content_sha256 excluding `created_at`, `manifest_sha256`, `path`). Never reads the claim. |
| aggregate report | `spot.stage02_run_manifest_verification.v1` | binds `verdict=="admit"`, `n_failed==0`, `generator_is_not_verifier==true`, `topology_complete`, `release_admissible`, `admission.status=="admitted"`, `manifest_sha256 == manifest_sha256_recomputed` |
| native bundles + ranking files | `records:[{target_id, arm_value, evaluable, rank}]` | reads `arm_value` **from here** — the authoritative value |
| **bridge** | `spot.stage02_stage3_bridge.v1` | re-hashes `bridge_sha256`; takes **identity + modality** from here |
| **bridge report** | (your separate bridge verifier) | admits the bridge from **this**, never from the bridge itself |

### The rule that makes the bridge safe

**The bridge may ADD facts the native bytes lack. It may never CHANGE a fact the native bytes
already state.**

Stage-3 takes `target_id_namespace` and `observed_perturbation_modality` from the bridge —
they exist nowhere else. But it re-checks **every `arm_value` against the native ranking file**
and refuses any disagreement. And it never trusts a serialized modulation: it **re-derives the
sign** from the native `arm_value` + `evaluable` and refuses if the bridge disagrees.

We do not bind `stage3_inputs` — your source defines it but the generated report does not emit
it. We bind the four paths explicitly.

## Two things we need from you

1. **Generated bytes.** A complete run root: `OUT/{direct,temporal,pathway}/<bundle-dirs>/`
   plus the root artifacts plus a **staged Stage-1 v3 release file** (the manifest embeds
   `stage1_v3_release` as a block; we must re-hash the actual file against that pin). Fixture
   scores are fine — we need the **shape**, hash-bound and admitted.
2. **The pathway lane fails open.** Your pathway verifier currently admits resealed
   target/modulation fields. We will not consume bytes admitted by a fail-open gate — an ADMIT
   from a gate that cannot refuse carries no information. Pathway contributes **zero** to our
   bundle until it fails closed. (Direct + temporal are unaffected; we verified that admission
   chain passes on your real bytes.)

## Where we agree already

Your `CTX_FORBIDDEN` — a pathway context may never carry `arm_value`, `rank`, `target_id`,
`observed_perturbation_modality`, `supported`, `phenocopy_claim` — is **exactly** our pathway
firewall, reached independently. We honour it verbatim. Pathway is context: it joins to
MEASURED rows via exact `leading_edge` typed identity, carries `pathway_id`/`source`/`coverage`/
`convergence` refs on the association, and **never sources a drug edge**.

## The sign rule we apply to your rows

Bound from your `disposition.desired_modulation` (`SIGN_EPS = 1e-9`, your `config.py:186`).
`arm_value` arrives **pre-oriented** to its arm's `desired_change` — we do not re-orient it.

- `arm_value > +ε` → knockdown supported the desired change → **inhibition is
  observed-compatible**, rankable as supported, labelled `putative_crispri_phenocopy`
- `arm_value < −ε` → knockdown opposed it → **an inhibitor is OPPOSED** (it phenocopies the
  *undesired* effect; kept and flagged, never ranked). An **agonist is an untested inverse
  hypothesis** — never a phenocopy, never supported. CRISPRi never tested activation.
- `|arm_value| ≤ ε` → `no_direction_evidence`, stated
- not evaluable → `not_evaluated`, stated

A drug inhibiting a **protein** is not CRISPRi silencing a **transcript**. Every edge carries
`evidence_relation` as a FIELD, not a caveat in prose.
