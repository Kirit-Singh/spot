Source-access & scoring reproduction trace

A recorded record of two things: (1) that the cells behind the spot · Stage-1 CD4 map originate in the public CZI Virtual Cells Platform dataset, verified by resolving the dataset and range-reading one split; and (2) that the continuous per-cell scoring is deterministic and reproduces per-barcode. This is not an end-to-end dataset reconstruction — it does not rebuild the 396,000-cell embedded object across all twelve splits; that object is fetched, pinned, from Hugging Face. Recorded run — 2026-07.

1 · Install

pip install vcp-cli then pip install 'vcp-cli[data]' → vcp-cli 0.54.1. vcp --help and vcp version run; the data command group activates with the [data] extra (boto3/s3transfer).

2 · Locate the dataset

vcp data search "Primary Human CD4+ T Cell Perturb-seq" resolves it under namespace billion-cell-project, published as 12 per-donor × per-condition AnnData splits (D1–D4 × Rest/Stim8hr/Stim48hr), each with its own dataset ID. vcp data describe … --full confirms provenance:

external_id : marson_D1_Rest.assigned_guide      (D1 Rest id 6946b5261d32b0e84ba87057)
assay       : 10x Flex
asset       : s3://genome-scale-tcell-perturb-seq/marson2025_data/D1_Rest.assigned_guide.h5ad

Two domains were allowlisted (both approved during the run): the data API cosmic-shepherd.prod-vcp.prod.czi.team and the public S3 bucket.

3 · Confirm the NTC pool (one split, range-read — not a full fetch)

The full dataset is 1.84 TB across 32 files; each per-split h5ad is 118–173 GB (all cells for that split — targeting + NTC). Nothing at that scale was downloaded. Only a metadata check and a single-split range-read were performed:

  • Downloaded (real, with md5): data_sharing_readme.md, the sample_metadata + sgRNA_library_metadata + DE_stats supplementary CSVs, and the D1 Rest .jsonld record → vcp_dl/.
  • Remote range-read (no full download), ONE split only: opened the 142 GB D1_Rest.assigned_guide.h5ad over s3fs + h5py and read only .obs/guide_type (~73 s, a few MB of range requests): 3,074,496 total cells, of which 76,634 (2.5%) are non-targeting — confirming the public NTC pool the workbench draws from. The other eleven splits were not read; the 396,000-cell object is not reconstructed here.

4 · Scoring reproducibility (continuous, per-barcode)

What is scored. The pinned, HF-published ntc_clustered.h5ad (a balanced NTC subsample — 33,000 cells per donor×condition = 396,000, all non-targeting) is scored against the panels with score_genes (panel mean minus expression-bin-matched control mean). The outputs are 12 continuous program scores per cell — no permutation, no p/q, no FDR, no categorical call.

Determinism. SEED=12345, control-gene indices sorted before summation, PYTHONHASHSEED=0. Run twice, the emitted 40,000-cell display overlay hashes identically at the per-barcode level.

Frozen per-barcode gate (verify_reproduce.py) — a canonical sorted table of barcode, cluster, condition, donor and the 12 rounded scores:

Input pin. ntc_clustered.h5ad is fetched from Hugging Face KiritSingh/spot-CD4-Marson at the immutable revision e5fcf98b… and its SHA-256 (2edc6d31…50e43) is verified before scoring. reproduce.sh fetches that pinned object and verifies the hash — it does not reconstruct the dataset through VCP. Not run here: no scVI re-embedding (the embedding/Leiden are loaded from the cached object); no download of any full split matrix.
