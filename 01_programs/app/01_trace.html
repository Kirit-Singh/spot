VCP reproducibility trace

A recorded, end-to-end test that the cells behind the spot · Stage-1 CD4 map come from the public CZI Virtual Cell Platform dataset via the vcp CLI, and that the scoring is deterministic and reproduces. Executed on a compute host through Claude Science, independently re-verified end-to-end (fresh vcp-cli install + S3 range-read + gate re-run). Recorded verification run — 2026-07-10 10:49 EDT.

1 · Install

pip install vcp-cli then pip install 'vcp-cli[data]' → vcp-cli 0.54.1. vcp --help and vcp version run; the data command group activates with the [data] extra (boto3/s3transfer).

2 · Locate the dataset

vcp data search "Primary Human CD4+ T Cell Perturb-seq" resolves it under namespace billion-cell-project, published as 12 per-donor × per-condition AnnData splits (D1–D4 × Rest/Stim8hr/Stim48hr), each with its own dataset ID. vcp data describe … --full confirms provenance:

external_id : marson_D1_Rest.assigned_guide      (D1 Rest id 6946b5261d32b0e84ba87057)
assay       : 10x Flex
asset       : s3://genome-scale-tcell-perturb-seq/marson2025_data/D1_Rest.assigned_guide.h5ad

Two domains were allowlisted (both approved during the run): the data API cosmic-shepherd.prod-vcp.prod.czi.team and the public S3 bucket.

3 · Fetch

The full dataset is 1.84 TB across 32 files; each per-split h5ad is 118–173 GB (all 2.2M+ cells for that split — targeting + NTC, not NTC-only). Too large to download in full, so the smallest sufficient artifacts were fetched instead:

  • Downloaded (real, with md5): data_sharing_readme.md, the sample_metadata + sgRNA_library_metadata + DE_stats supplementary CSVs, and the D1 Rest .jsonld record → vcp_dl/. Checksums in the trace.
  • Remote range-read (no full download): opened the 142 GB D1_Rest.assigned_guide.h5ad over s3fs + h5py and read only .obs/guide_type (~73 s, a few MB of range requests): 3,074,496 total cells, of which 76,634 (2.5%) are non-targeting — the public NTC pool the workbench draws from.

4 · Reproducibility

Provenance link. The cached ntc_clustered.h5ad is a balanced NTC subsample — exactly 33,000 cells per donor×condition = 396,000, all non-targeting; for D1/Rest that's 33k sampled from the 76,634 public NTC cells confirmed above. Barcodes carry the lane/donor tag.

Determinism (byte-exact). Run twice at SEED=12345 → max|diff| = 0.0; the per-cell nomenclature calls hash identically (MD5 a0a3f1a8ad62c9e5766c7c42acf9a67f). A different seed (999) shifts scores as expected (control-gene draw).

Reproduction at the committed N_PERM=500, seed 12345 matches the committed run on every class:

N_PERM sensitivity. A reduced N_PERM=100 spot-check does not reproduce the fractions (no-call jumps to 96.6%) — the empirical-p floor (1+0)/(1+N_PERM) is larger with fewer permutations, so the FDR floor is N_PERM-sensitive and reproduction requires the committed 500. Determinism (byte-exact) holds at any N_PERM.

Not run: no scVI re-embedding (the embedding/Leiden are loaded from the cached object); no download of any full split matrix.
